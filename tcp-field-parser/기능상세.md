# TCP/IP 전문 파서

TCP/IP 고정 길이 전문을 필드 정의에 따라 파싱하고 검증하는 도구.

## 주요 기능

- **멀티 탭 워크스페이스** — 여러 전문 스펙을 탭으로 관리. 탭에 이름, 설명, 색상 그룹 부여 가능
- **필드 정의 DSL** — 한 줄에 하나의 필드를 정의하는 간단한 문법
- **요청/응답 분리 파싱** — 탭마다 요청 전문과 응답 전문을 별도 입력·파싱
- **고정값 검증** — 필드에 기대값을 지정하면 실제 값과 자동 비교
- **서브필드 참조** — `{@탭라벨}` 문법으로 다른 탭의 필드 정의를 중첩 파싱
- **잔여 전문 표시** — 정의된 필드 이후 남은 바이트를 별도 행으로 표시
- **전문 이력** — 파싱 결과를 탭별로 저장하고 이력 모달에서 재조회
- **가져오기/내보내기** — 탭 설정을 JSON으로 내보내고 가져오기 (파일 드래그앤드롭 지원)
- **다크/라이트 모드** — `prefers-color-scheme` 자동 감지

## 필드 정의 문법

```
길이만              →  10
이름:길이           →  거래코드:4
이름:길이=고정값    →  전문구분:2=01
이름:길이{@탭라벨}  →  서브필드 참조 (해당 탭의 정의로 재귀 파싱)
이름:길이{@탭라벨}=고정값  →  참조 + 고정값 검증 조합
```

- `#`으로 시작하는 줄은 주석으로 무시
- 길이는 **바이트** 기준 (EUC-KR: 한글 2바이트, UTF-8: 한글 3바이트)

## 인코딩

탭마다 `EUC-KR` / `UTF-8` 선택 가능. 바이트 슬라이싱 및 길이 계산이 인코딩에 따라 달라짐.

## 데이터 저장

IndexedDB(`tcpParser`, version 3) 사용:

| 스토어 | keyPath | 인덱스 | 용도 |
|--------|---------|--------|------|
| `tabs` | `id` | `order` | 탭 전체 상태 영속 |
| `msgLog` | `logId` | `savedAt`, `tabId` | 전문 이력 |

탭 그룹 접힘 상태는 `localStorage`(`tcpParserGroupCollapse`)에 별도 저장.
